## What You're Coming To

The AWS Well-Architected Framework is AWS's set of cloud architecture best practices, distilled from thousands of real customer workload reviews. It gives engineering, security, and finance teams a **shared vocabulary** for talking about trade-offs, and it catches risk *before* it becomes an incident.

The framework is organized into **six pillars**:

1. **Operational Excellence** — how you run, monitor, and improve systems
2. **Security** — how you protect data, systems, and access
3. **Reliability** — how you recover from failure and handle changing demand
4. **Performance Efficiency** — how you match resources to the workload
5. **Cost Optimization** — how you avoid unnecessary spend
6. **Sustainability** — how you minimize environmental impact

One pillar per module. Each pillar has a hands-on lab. The day ends with a capstone lab where you run a lightweight Well-Architected Review on a sample workload.

## What You Should Already Know

This is an **introductory** workshop, but you'll get more out of it if you're comfortable with:

- **What "the cloud" is** — you don't need to be an AWS expert, but you should know that AWS rents compute, storage, and databases as a service
- **Roughly what these three services do:**
  - **EC2** — virtual servers
  - **S3** — object storage (buckets and files)
  - **IAM** — who can do what in an AWS account
- **Reading JSON** — a couple of labs show short JSON policies; you don't need to write any
- **Basic networking concepts** — what a firewall is, what a DNS record is

If any of the above is unfamiliar, don't skip the class — just know that the first hour will feel denser. The decks include glossaries and the labs define every acronym before using it.
