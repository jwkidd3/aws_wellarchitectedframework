Objective

Write a runbook — a step-by-step procedure — clear enough that another engineer, woken at 3 AM and never having seen the system, could execute it successfully.

Then identify which steps you would automate first.

Terms You'll See

  • ALB (Application Load Balancer) — AWS service that distributes incoming HTTP requests across multiple servers
  • ASG (Auto Scaling Group) — a collection of EC2 servers that grows and shrinks automatically based on load
  • Aurora — AWS's managed MySQL/PostgreSQL-compatible database
  • Connection pool — a reusable set of database connections an app keeps open; if all are busy, new requests wait
  • 5xx error — HTTP status codes 500–599, meaning the server failed to handle the request
  • PagerDuty — a third-party on-call alerting service

Scenario

  • Customer-facing API on an EC2 Auto Scaling group behind an Application Load Balancer
  • Backed by an Aurora MySQL database
  • The alert just fired:
    api-prod / 5xx error rate > 5% for 10 min
  • You've seen this before — it's usually Aurora's connection pool being exhausted because a stuck application thread is holding connections open

Your Runbook Must Include

  1. Title & symptom — what alert triggered this runbook
  2. Pre-checks — quick checks to confirm this is the right runbook
  3. Mitigation steps — ordered, specific, with expected outcome at each step
  4. Verification — how you know it worked
  5. Rollback — what to do if a step makes things worse
  6. Escalation — who to page and when
  7. Post-incident — what to capture for the retrospective

Constraints

  • No step may say "investigate" or "check things" — be concrete
  • Every command should be copy-pasteable (placeholders like <instance-id> are fine)
  • Maximum one page

Then…

Mark each step:

  • [A] — automate this next quarter
  • [H] — keep human-driven

Be ready to defend your choices.

Discussion

  • Which step was hardest to write concretely? Why?
  • What CloudWatch dashboard would make this runbook faster?
  • If this fires twice a month, what should change in the system — not just the runbook?
