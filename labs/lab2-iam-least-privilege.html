Lab 2

IAM Least Privilege Review

Pillar: Security

Objective

Read a sample IAM policy attached to an application role, identify least-privilege violations, and rewrite it so the application can do its job — and nothing else.

Scenario — app-orders-processor

  • Reads order events from one SQS queue:
    arn:aws:sqs:us-east-1:111122223333:orders-incoming
  • Writes to one DynamoDB table:
    arn:aws:dynamodb:us-east-1:111122223333:table/orders
  • Writes to its own CloudWatch log group
  • Runs on EC2 in 111122223333 / us-east-1

The Current Policy

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AppAccess",
      "Effect": "Allow",
      "Action": "*",
      "Resource": "*"
    },
    {
      "Sid": "ExtraStuff",
      "Effect": "Allow",
      "Action": [
        "s3:*", "iam:PassRole",
        "dynamodb:*", "sqs:*", "logs:*"
      ],
      "Resource": "*"
    }
  ]
}

Tasks

  1. List at least 5 violations. Consider action wildcards, resource wildcards, unneeded privileges, escalation risks, cross-region/account exposure.
  2. Rewrite the policy — scoped by action and resource ARN.
  3. Bonus: add a Condition restricting the role to us-east-1.

Hints

  • DynamoDB: GetItem, PutItem, UpdateItem, BatchWriteItem
  • SQS: ReceiveMessage, DeleteMessage, GetQueueAttributes
  • Logs: CreateLogStream, PutLogEvents — scope to log group ARN
  • Region condition key: aws:RequestedRegion

Sample Solution

{
  "Statement": [
    {
      "Sid": "ReadOrderQueue",
      "Effect": "Allow",
      "Action": ["sqs:ReceiveMessage","sqs:DeleteMessage","sqs:GetQueueAttributes"],
      "Resource": "arn:aws:sqs:us-east-1:111122223333:orders-incoming"
    },
    {
      "Sid": "WriteOrdersTable",
      "Effect": "Allow",
      "Action": ["dynamodb:GetItem","dynamodb:PutItem","dynamodb:UpdateItem","dynamodb:BatchWriteItem"],
      "Resource": "arn:aws:dynamodb:us-east-1:111122223333:table/orders"
    },
    {
      "Sid": "WriteAppLogs",
      "Effect": "Allow",
      "Action": ["logs:CreateLogStream","logs:PutLogEvents"],
      "Resource": "arn:aws:logs:us-east-1:111122223333:log-group:/app/orders-processor:*"
    }
  ]
}

Discussion

  • How would you detect a policy like the original before it reaches production?
  • Which AWS services help enforce least-privilege?
    IAM Access Analyzer · SCPs · Config rules
