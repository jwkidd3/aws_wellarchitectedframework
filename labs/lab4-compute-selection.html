Lab 4

Compute Selection

Pillar: Performance Efficiency

Objective

For three workloads, pick a compute option (Lambda, Containers, or EC2) and justify it against traffic shape, runtime constraints, and operational cost.

No single right answer — defend your reasoning.

Terms You'll See

  • EC2 (Elastic Compute Cloud) — virtual machines; you manage the OS and the software on them
  • Container — a self-contained package of code + dependencies that runs the same anywhere
  • ECS / EKS / Fargate — AWS services for running containers (Fargate = no servers to manage)
  • Lambda — "serverless" functions; you provide code, AWS runs it when an event fires, and you pay per invocation
  • Spot instance — discounted EC2 capacity (up to ~90% off) that AWS can reclaim with 2 minutes' notice
  • Checkpoint — periodically saving progress so a job can resume after interruption
  • GPU (Graphics Processing Unit) — specialized hardware for highly parallel math, used in ML and scientific computing

Decision Aids

  • Lambda shines for: short bursty work, event-driven, unpredictable load, minimal operations overhead
  • Containers shine for: long-running services, complex runtimes, portability, steady or predictable burst load
  • EC2 shines for: specialized hardware (GPU, huge RAM), very long jobs, software licensed to a specific host, or when you genuinely need OS-level control

Workload A — Image Thumbnail Generator

  • Triggered each time a user uploads a photo to S3
  • ~50,000 events/day, very bursty (spikes in the evenings)
  • Each invocation runs 2–8 seconds, mostly CPU work
  • Must scale instantly; idle most of the night

Workload B — Internal Analytics API

  • Steady ~200 requests per second, 24×7
  • Custom Python service with native dependencies (NumPy, pandas, a C library)
  • Average request 150 ms; long tail of 2-second requests
  • Team has Kubernetes experience and existing CI for container images

Workload C — Genomic Sequence Aligner

  • Batch job, 6–12 hours per execution
  • Requires 192 GB RAM and a specific GPU instance family
  • Runs 3–4 times per week, scheduled in advance
  • Cost-sensitive; interruption is recoverable from a checkpoint

For Each Workload

  1. Compute choice — Lambda, Containers, or EC2
  2. Specifics — instance type / Lambda memory / Fargate vCPU as relevant
  3. Why not the others — one sentence per rejected option
  4. Pillar interactions — does your choice change Cost, Reliability, or Sustainability outcomes?

Console Exploration (read-only)

Navigate only — do not launch, stop, or modify anything.

  1. Search for EC2 → Instances. Notice the instance types running.
  2. Search for Lambda → Functions. Open any function and look at the Configuration tab — memory, timeout, trigger.
  3. Search for ECS → Clusters. Browse any clusters and their tasks.
  4. Compare: same application could live in any of these — the choice is about fit, not capability.

Discussion

  • Which workload had the most divided answers?
  • Would Spot instances be appropriate for Workload C? For A?
  • What would have to change to make Lambda the right answer for Workload B?
