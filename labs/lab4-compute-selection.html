Lab 4

Compute Selection

Pillar: Performance Efficiency

Objective

For three workloads, pick a compute option (Lambda, Containers, or EC2) and justify it against traffic shape, runtime constraints, and operational cost.

No single right answer — defend your reasoning.

Decision Aids

  • Lambda — short bursty work, event-driven, unpredictable load, minimal ops
  • Containers — long-running services, complex runtimes, portability, steady or predictable load
  • EC2 — specialized hardware, GPU, very long jobs, host-level licensing, OS control

Workload A — Image Thumbnail Generator

  • Triggered when a user uploads to S3
  • ~50,000 events/day, very bursty (evening spikes)
  • Each invocation 2–8 seconds, mostly CPU
  • Must scale instantly; idle most of the night

Workload B — Internal Analytics API

  • Steady ~200 req/sec, 24×7
  • Custom Python with native deps (NumPy, pandas, C library)
  • Avg 150 ms; long tail to 2 sec
  • Team has Kubernetes experience & CI for images

Workload C — Genomic Sequence Aligner

  • Batch job, 6–12 hours per execution
  • Requires 192 GB RAM and a specific GPU instance family
  • Runs 3–4 times/week, scheduled in advance
  • Cost-sensitive; interruption recoverable from checkpoint

For Each Workload

  1. Compute choice — Lambda, Containers, or EC2
  2. Specifics — instance type / Lambda memory / Fargate vCPU
  3. Why not the others — one sentence per rejected option
  4. Pillar interactions — Cost, Reliability, Sustainability impact

Discussion

  • Which workload had the most divided answers?
  • Spot instances — appropriate for C? For A?
  • What would have to change to make Lambda right for B?
