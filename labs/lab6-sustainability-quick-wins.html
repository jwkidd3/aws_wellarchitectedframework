Lab 6

Sustainability Quick Wins

Pillar: Sustainability

Objective

Review a small reference architecture, list sustainability improvements, and show how each one also improves at least one other pillar.

The Architecture (B2B SaaS Analytics)

  • Region: us-east-1 only (chosen years ago, no reason)
  • Web: 12 × m5.xlarge, fixed 24×7, avg CPU 12%
  • API: 8 × c5.2xlarge, fixed 24×7, avg CPU 18%
  • Workers: 6 × m5.2xlarge, nightly batch ~2h, idle 22h

The Architecture (cont.)

  • DB: Aurora 2 × db.r5.4xlarge + identical staging clone 24×7
  • Storage: 45 TB S3 Standard, 70% not accessed in 6+ months, no lifecycle
  • Logs: CloudWatch with no retention
  • CDN: none — every request hits ALB, including static JS

Tasks

  1. List 6–8 sustainability improvements. For each:
    • What you'd change
    • Which other pillar it benefits
    • Effort: S / M / L
  2. Pick the top 3 to do this quarter — defend the order
  3. Identify one improvement that would worsen another pillar if done carelessly

Hints — Look For

  • Idle capacity, oversized instances
  • Missing auto-scaling
  • Missing storage tiering / lifecycle
  • Missing CDN
  • Missing log retention
  • Region choice for non-latency-sensitive work
  • Older instance generations (Graviton)

Discussion

  • Did your top 3 align with what Lab 5 (Cost) would have prioritized? Why?
  • For the worker tier — do those workers need to be EC2 at all?
  • "Sustainability is just Cost Optimization with a green hat." Where does that break down?
