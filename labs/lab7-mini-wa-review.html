Capstone Lab · 45 min

Lab 7

Mini Well-Architected Review

All six pillars

Objective

Run a lightweight Well-Architected Review against a sample workload, assign risk levels across all six pillars, and produce a prioritized remediation roadmap.

Work in pairs or small groups. The point is the conversation the framework forces.

The Workload — "OrderFlow"

  • B2B order-management SaaS, ~3 years old
  • Team inherited it last quarter
  • Single AWS account, us-east-1 only
  • ALB → 6 × m5.large EC2, fixed (no Auto Scaling), Node.js
  • Aurora MySQL, 1 writer + 1 reader, single AZ

OrderFlow (cont.)

  • S3 bucket for customer uploads, no lifecycle
  • Jenkins on EC2 deploys via SSH + shell scripts
  • Default CloudWatch metrics, no custom dashboards
  • 14 IAM users, several with AdministratorAccess
  • Nightly Aurora snapshot, 7-day retention, never tested

OrderFlow (cont.)

  • One runbook in a Google Doc, last updated 14 months ago
  • Cost: ~$11k/month, no Savings Plans, no tagging policy
  • No CDN — static assets served from EC2
  • Customers report intermittent slow page loads at peak

For Each Pillar

  1. Note 2–3 findings — specific issues
  2. Assign a risk: High / Medium / None
  3. Pick the highest-impact fix for that pillar

Six pillars: OpEx · Security · Reliability · Performance · Cost · Sustainability

Then Across All Pillars

  1. Roadmap — top 5 items in order, with effort (S/M/L)
  2. Cross-pillar wins — fixes that help two or more pillars
  3. Trade-offs — fixing one pillar hurts another, how you'd negotiate

Deliverable — One-Page Summary

  • The 3 things you'd do this sprint
  • The 2 things you'd do this quarter
  • The 1 thing leadership needs to know about now

Debrief

  • Where did your team disagree most — priority or facts?
  • How would answers change if OrderFlow were Tier-0 financial software?
  • Which pillar was hardest to assess from this description?
