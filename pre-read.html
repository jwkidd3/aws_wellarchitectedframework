Workshop Pre-Read

AWS Well-Architected Best Practices

1 day · Introductory · 09:00 – 16:00

Please read this before class — about 10 minutes.

What You're Coming To

  • The AWS Well-Architected Framework is AWS's set of cloud architecture best practices, distilled from thousands of real customer workload reviews
  • It gives engineering, security, and finance teams a shared vocabulary for trade-offs
  • Catches risk before it becomes an incident
  • Organized into six pillars

The Six Pillars

1. Operational Excellence
Run & improve systems
2. Security
Protect data & access
3. Reliability
Recover from failure
4. Performance Efficiency
Use resources well
5. Cost Optimization
Avoid waste
6. Sustainability
Reduce footprint

One pillar per module. Each pillar has a hands-on lab. The day ends with a capstone lab where you run a lightweight review on a sample workload.

Official Reference

  • AWS Well-Architected Framework homepage:
    https://aws.amazon.com/architecture/well-architected/

You don't need to read this cover-to-cover. Skim the pillar overviews so the names feel familiar on day one.

What You Should Already Know

This is an introductory workshop, but you'll get more out of it if you're comfortable with:

  • What "the cloud" is — you don't need to be an AWS expert, but you should know that AWS rents compute, storage, and databases as a service
  • Roughly what these three services do:
    • EC2 — virtual servers
    • S3 — object storage (buckets and files)
    • IAM — who can do what in an AWS account
  • Reading JSON — a couple of labs show short JSON policies
  • Basic networking concepts — what a firewall is, what a DNS record is

If Some of That Is Unfamiliar…

  • Don't skip the class — just know the first hour will feel denser
  • The decks include glossaries and the labs define every acronym before using it
  • Your instructor and the people around you are resources — ask questions

What You Do NOT Need

  • An AWS account of your own — a shared class account is provided
  • Any software installed on your laptop — everything runs in the browser
  • Prior Well-Architected Framework experience

What To Bring

  • A laptop with a modern browser (Chrome, Edge, Firefox, or Safari)
  • Pen and paper — most labs are analysis exercises worked through on paper before discussing
  • Questions about workloads you actually own — the framework is most useful when applied to real systems you know

On the Day

  • Your instructor will hand out AWS Console credentials at the start of the day
  • The login URL and your username (user1, user2, …) will be covered in Lab 1
  • You'll use the AWS Console for navigation only — no creating, editing, or deleting resources
  • Labs are mostly discussion and analysis; console exploration is a few minutes per lab
