Pillar 5

Cost Optimization

Avoid unnecessary cost — at every layer.

FinOps Mindset

FinOps = Financial Operations: treating cloud cost as a shared engineering + finance responsibility.

  • Engineers make the decisions that drive cost, so engineers need the cost data
  • Cost reviews belong in sprint planning, not once-a-quarter panic
  • Visibility → accountability → optimization, in that order
  • Celebrate cost reductions like feature launches

How AWS Billing Actually Works

  • Metered, pay-as-you-go — you're billed for what you use, typically by the second or hour
  • No up-front cost to spin something up; no refund for idle time
  • Charges come from many dimensions: compute hours, storage GB, data transfer GB, API calls, etc.
  • Data transfer out to the internet costs real money; data in is usually free
  • Even "small" services (NAT Gateway, idle load balancers, old snapshots) can quietly add hundreds of dollars/month

Know What You're Spending

  • Cost Explorer — AWS's built-in tool to visualize spend by service, account, or tag
  • Budgets — email/SMS alerts when cost or usage crosses a threshold
  • Cost Anomaly Detection — machine-learning alerts for unusual cost spikes
  • Tagging — label every resource with metadata like env=prod, owner=team-a, cost-center=42. You can't optimize what you can't attribute.

Pricing Models

  • On-Demand — full price, pay per hour/second, no commitment
  • Savings Plans — commit to a $/hour spend for 1 or 3 years → up to ~72% off
  • Reserved Instances (RI) — older commitment model, still used for RDS
  • Spot — unused AWS capacity at up to ~90% off, but AWS can reclaim it with 2 minutes' notice. Great for batch jobs and stateless workers that can restart.

Right-Sizing

Right-sizing = matching the instance size to what the workload actually needs.

  • Compute Optimizer — AWS service that recommends smaller/cheaper instance types based on real usage
  • Look for instances sustained under ~20% CPU
  • Don't forget storage: upgrade gp2 → gp3 EBS volumes (cheaper and faster), delete unattached volumes and old snapshots

Tagging Strategy

  • Tags are key=value labels on resources. They're free, and they're the foundation of cost allocation.
  • Minimum required tags:
    • env — prod, staging, dev
    • owner — team or individual
    • cost-center — for chargeback
    • app — which application
  • Enforce with SCPs or Config rules — resources without required tags get blocked or flagged
  • Activate tags as cost allocation tags in the Billing console so Cost Explorer can group by them

Data Transfer — The Silent Killer

  • Data transfer charges show up in tiny amounts everywhere and add up fast
  • Free: data in; most traffic within the same AZ
  • Cheap: traffic between AZs in the same region (~$0.01/GB each way)
  • More expensive: cross-region traffic (~$0.02/GB)
  • Most expensive: data out to the internet ($0.05–$0.09/GB)
  • Use VPC endpoints to reach AWS services privately without NAT Gateway data-processing fees
  • Use CloudFront — outbound via CloudFront is cheaper than direct from EC2

Match Supply to Demand

  • Auto Scaling — add servers at peak, remove them at quiet times
  • Schedule — turn dev/test environments off nights and weekends
  • Serverless (Lambda, Fargate) — pay only when code actually runs
  • Idle infrastructure is the most expensive infrastructure
